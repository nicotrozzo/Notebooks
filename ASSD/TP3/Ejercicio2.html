Conversor Analógico a Digital: Sigma Delta¶

Se busca realizar una simulacin del proceso completo de discretizacin y cuantizacin que constituyen al pasaje del dominio analgico a digital, empleando para ello un muestreo y un conversor siguiendo la arquitectura de un ADC Sigma Delta. Se analizarn los aspectos particulares que hacen a este conversor, y las consecuencias que dichos detalles implican como mejoras sobre otros tipos de conversores.

Objetivo¶

El principal objetivo de la utilizacin de conversores $\Sigma \Delta$ es alcanzar una relacin de seal a ruido elevado en sistemas cuya calidad dependen de este parmetro, y por ello buscan que sea elevado. En particular, se sabe que un ADC de una $n$ cantidad de bits tiene una SQNR en dB:

\begin{equation} SQNR = 6.02dB \cdot n + 1.76dB \end{equation}

La modulacin del $\Sigma \Delta$ produce un efecto llamado Noise Shaping que en conjunto con el oversampling, permiten obtener una misma cantidad $n$ de bits resultantes, pero con un SQNR mucho menor. Por un lado el oversampling permite que la potencia del ruido que depende nicamente de la cantidad de bits se distribuya en un espectro mucho ms amplio, lo cual produce que la potencia contenida en la regin til de frecuencias que cumplen el criterio de Nyquist se vea disminuida. En el siguiente grfico se ve la misma potencia distribuida en el espectro de frecuencias disponible habiendo muestreado, no se grafica el espectro negativo, pero parte de la potencia se ve contenida all.

Desde un punto de vista probabilstico, la seal de entrada podra ser una variable aleatoria continua con un determinado desvo estandar, para el cual es necesario una determinada resolucin para ganantizar que se acepte el rango dinmico esperado. No obstante, en los conversores $\Sigma \Delta$, como se cuantiza constantemente una seal diferencial como resta de la entrada y la seal que se pretende obtener, la varianza de tal diferencia disminuye mucho y con ella el rango dinmico. Lo cual permite disminuir el requisito de resolucin, es decir, la cantidad de bits.

Entonces, con la modulacin $\Sigma \Delta$, es posible realizar una cuantizacin con un slo bit, donde la SQNR se ve aumentada. Este proceso se explicar con ms detalle en su respectiva seccin. Finalmente, digitalmente la seal de 1 bit del $\Sigma \Delta$ debe ser filtrada con un pasabajos para recuperar el espectro base y luego utilizar un downsampling, este efecto en conjunto se denomina Decimacin, y permitir obtener el resultado con una N cantidad de bits a una frecuencia de Nyquist.

import numpy as np

def noise(power: float):
    """ Generador de ruido gaussiano.
        @param power: Potencia del ruido
        @return: Funcin f(t)
    """
    def wrapp(t: float):
        return np.random.normal(scale=np.sqrt(power))
    return wrapp

def dc(a: float):
    """ Generador de continua.
        @param a: Valor DC
        @return: Funcin f(t)
    """
    def wrapp(t: float):
        return a
    return wrapp

def triangle(a: float, f: float, d: float, c: float):
    """ Generador de seal triangular
        @param a: Amplitud
        @param f: Frecuencia
        @param d: Duty, simetra...
        @param c: Nivel de continua
        @return: Funcin f(t)
    """
    def wrapp(t: float):
        T = 1 / f
        _t = t % T
        if _t <= (d * T / 100):
            return -a + 2 * a * (_t / (d * T / 100)) + c
        else:
            return a - 2 * a * ((_t - d * T / 100) / ((100 - d) * T / 100)) + c
    return wrapp

def senoidal(a: float, f: float, p: float, c: float):
    """ Generador de seal senoidal.
        @param a: Amplitud
        @param f: Frecuencia
        @param p: Fase en radianes
        @param c: Nivel de continua
        @return: Funcin f(t)
    """
    def wrapp(t: float):
        return a * np.sin(2 * np.pi * f * t + p) + c
    return wrapp

def square(a: float, f: float, d: float, c: float):
    """ Generador de seal cuadrada.
        @param a: Amplitud
        @param f: Frecuencia
        @param d: Duty del 0 al 100%
        @param c: Nivel de continua
        @return: Functin f(t)
    """
    if d < 0 or d > 100:
        raise ValueError("El duty cycle est fuera de rango.")
    def wrapp(t: float):
        T = 1 / f
        _t = t % T
        return (a if _t < (d * T / 100) else -a) + c
    return wrapp

¿Cómo uso la simulación?¶

En la siguiente celda de cdigo de Python tenemos la configuracin que nos puede permitir modificar algunos parmetros para poder observar diferentes condiciones de simulacin. Para mayor claridad, se puede dividir en 3 etapas o partes:

1. Configuración de la señal de entrada¶

En la celda previa de cdigo en Python se definieron algunos generadores de seales, se debe utilizar alguno de esos generadores configurados, indicando la frecuencia de entrada que se est utilizando y cuntos perodos se desean visualizar en el proceso. Para ello, especificar:

  • fin: Frecuencia de la seal de entrada, su valor fundamental en caso de poliarmnica.
  • periods: Cantidad de perodos a mostrar, en caso de no tener una seal peridica, poner fin y periods para mostrar el tiempo deseado.
  • x_t: Cargar un generador de seal con sus parmetros, esto es, llamar a una de las funciones con los parmetros.

2. Configuración del ADC¶

En este caso se configura cuntos bits se quieren en la salida digital, y cul es la frecuencia mxima de entrada que llamamos como frecuencia de paso. La idea es que en funcin de esto, la simulacin calcula automticamente la frecuencia del criterio de Nyquist, y el valor del oversampling. Entonces, especifique:

  • fp: Frecuencia mxima de entrada
  • n: Cantidad de bits
  • aa: Atenuacin en banda de rechazo
  • vref: Tensin de referencia del ADC, marca los lmites. $|V_i| < V_{REF}$.

Nota: Se cambia la tensin de referencia del DAC del $\Sigma \Delta$ que se mueve entre +/- $V_{REF}$.

3. Configuraciones generales de simulación¶

Finalmente, esto es para configurar aspectos de la simulacin inherentes al funcionamiento de la misma.

  • count: Cantidad de puntos entre muestras del tiempo discreto, esto es a modo didctico o ilustrativo, para poder tener una percepcin de continuidad cuando se contrastan seales de tiempo discreto y tiempo continuo.

class ContinuousSignal:
    """ El propsito de ContinuousSignal nuevamente, es meramente ilustrativo. Al fin y al cabo todo lo que trabaja un
        lenguaje de programacin son seales discretas, entonces nos armamos esta clase para tener los puntos de la seal
        continua y simular el contraste continua y discreta.
        Bsicamente lo que hace es permite calcular los puntos de muestreo, y agrega entre ellos una cantidad dada
        de puntos de la seal, todos en total hacen a la continua, pero algunos hacen al muestreo. Lo hacemos as
        para garantizar algo de compatibilidad en el manejo de puntos a lo largo de la simulacin.
        Espero no estar sobrecomplicando. 
        Saludos.
    """
    
    @staticmethod
    def build_from_continuous(times: np.array, values: np.array, fs: float, samples: int, count: int):
        """ Se agrega una factory a partir de datos ya procesados en el continuo para reutilizar funcionalidades, 
            pero sin usar uan funcin de la cual muestrear.
            Esto es para por ejemplo, cuando se proces despus de un sistema analgico como el FAA, y se quiere la 
            misma caracterizacin Continuo/Discreta.
            @param times: Espacio 'continuo' de tiempo
            @param values: Espacio de muestras
            @param fs: Frecuencia de muestreo
            @param samples: Cantidad de muestras discretas
            @param count: Sobrecarga de muestras para percibir un 'continuo' entre muestras discretas
            @return: Devuelve una ContinuousSignal, para que puedas ver su forma continua, y el resultado de su muestreo
        """
        if len(values) != len(times) or len(values) != (samples * (count + 1)):
            raise ValueError("Tamao de las muestras y su configuracin es imcompatible.")
            
        # Construyo la seal y la devuelvo
        _signal = ContinuousSignal(None, fs, samples, count)
        _signal.times = times
        _signal.values = values
        return _signal
    
    @staticmethod
    def build_from_discrete(values: np.array, fs: float, samples: int, count: int):
        """ Se agrega una factory a partir de datos ya procesados que sean discretos. Repite la funcionalidad anterior,
            pero refleja la forma continua del discreto haciendo un hold entre muestras.
            @param times: Espacio 'continuo' de tiempo
            @param values: Espacio de muestras
            @param fs: Frecuencia de muestreo
            @param samples: Cantidad de muestras discretas
            @param count: Sobrecarga de muestras para percibir un 'continuo' entre muestras discretas
            @return: Devuelve una ContinuousSignal, para que puedas ver su forma continua, y el resultado de su muestreo
        """
        if len(values) != samples:
            raise ValueError("Tamao de las muestras y su configuracin es imcompatible.")
            
        # Construyo la seal y la devuelvo
        _signal = ContinuousSignal(None, fs, samples, count)
        for i in range(_signal.length):
            _signal.times[i] = i * _signal.step
            _signal.values[i] = values[i // (_signal.count + 1)]
        return _signal
    
    def __init__(self, func: callable, fs: float, samples: int, count: int):
        """ Se construye una ContinuousSignal
            @param func: Funcin que permite obtener puntos en 'cualquier' valor continua.
            @param fs: Frecuencia de muestreo.
            @param samples: Cantidad de muestras a tomar.
            @param count: Cantidad de puntos adicionales entre muestras.
        """
        # Guardando los atributos o datos miembros de la clase
        self.fs = fs
        self.samples = samples
        self.count = count
        
        # Inicializando muestras internas
        self.length = samples * (count + 1)
        self.values = np.zeros(self.length)
        self.times = np.zeros(self.length)
        # Computando 'muestras' internas
        self.step = 1 / (fs * (count + 1))
        if func is not None:
            for i in range(self.length):
                self.values[i] = func(i * self.step)
                self.times[i] = i * self.step
    
    def downsample(self, n: int):
        """ Devuelve una nueva instancia ContinuousSignal que corresponde a la caracterizacin de la seal
            muestreada a una frecuencia inferior, realizando Downsampling con una relacin de fo = fi / n.
            @param n: Cantidad de veces que se reduce la frecuencia de sampling
            @return: Devuelve una ContinuousSignal
        """
        _fs = self.fs / n
        _samples = self.samples // n
        _count = self.count
        _values = [self.values[i * n * (_count + 1)] for i in range(_samples)]
        return ContinuousSignal.build_from_discrete(_values, _fs, _samples, _count)
    
    def continuous(self):
        """ Devuelve la informacin de tiempo y valor de la seal 'continua'.
            @return: (t, x)
        """
        return self.times, self.values
    
    def instantaneous_sampling(self):
        """ Se devuelve la seal continua del proceso de muestreo natural, es decir, corresponde al resultado
            de realizar un sample and hold sobre la seal correspondiente.
            @return: (t, x)
        """
        return (
            self.times,
            [self.values[(i // (self.count + 1)) * (self.count + 1)] for i in range(self.length)]
        )
    
    def discrete_sampling(self):
        """ Se devuelven los instantes y valores de muestreo, que quede claro que por discreto entendemos que
            los puntos devueltos corresponden a instantes pura y exclusivamente discretizados. Sera hablar de x(n).
            @return: (t_n, x_n)
        """
        return (
            [self.times[i * (self.count + 1)] for i in range(self.samples)], 
            [self.values[i * (self.count + 1)] for i in range(self.samples)]
        )

1. Filtro Antialias¶

Se realiza el filtrado con un filtro antialias, pasabajos, que limita el espectro de la seal de entrada al sistema para evitar que se produzca aliasing. Particularmente, el uso de oversampling con una frecuencia muy elevada permite aumentar la banda de transicin lo cual disminuye la pendiente requerida, por ende esta etapa puede ser conseguida con filtros de orden reducido. Esto resulta una ventaja por sobre las condiciones normales, que requieren hacer uso de filtro de mucha selectividad por el rango acotado de la banda de transicin cuando se toma una frecuencia en el orden establecido por el criterio de Nyquist.

En el caso de necesitar un filtro de primer orden, se utilizara una expresin sencilla como:

\begin{align} & H_{FAA}(s) = \frac{1}{1 + s \cdot R \cdot C} = \frac{1}{1 + \frac{s}{\omega_o}} \\ & H_{FAA}(w) = \frac{1}{1 + j \cdot \frac{\omega}{\omega_o}} \Rightarrow |H_{FAA}(\pi \cdot f_s)| = \frac{1}{\sqrt{1 + (\frac{f_s}{f_o \cdot 2})^{2}}} \Rightarrow |H_{FAA}(\pi \cdot f_s)| [dB] = -10 \cdot log(1 + (\frac{f_s}{f_o \cdot 2})^{2}) = -A_a \end{align}\begin{equation} f_o = \frac{f_s}{2} \cdot \frac{1}{\sqrt{10^{\frac{A_a}{10}} - 1}} \end{equation}

En el caso de necesitar un orden mayor, se propone hacer uso de Butterworth, pues el filtro no debe ser demasiado selectivo. En este sentido, tambin podra aprovecharse la baja selectividad para utilizar un filtro Gaussiano o Bessel, para disminuir la distorsin por fase no lineal, y tener un mejor control del retardo de grupo. En otros escenarios, limitarse a la frecuencia de Nyquist podra provocar que la alta selectividad nos impida hacer uso de un Bessel.

def first_order_pole(fs: float, aa: float) -> float:
    """ Se calcula la ubicacin del polo del sistema de primer orden para lograr
        atenuar por debajo de lo requerido en el lmite de la condicin de Nyquist.
        Creemos recomendable revisar luego que la ubicacin mantenga cierta coherencia
        con la ubicacin de la banda pasante, es decir est en una frecuencia posterior.
        @param fs: Frecuencia de muestreo
        @param aa: Atenuacin mnima en banda atenuante
        @return: Frecuencia del polo de primer orden que cumple requisitos
    """
    return (fs / 2) * ( 1 / np.sqrt(10 ** (aa / 10) - 1) )

def estimate_order(fp: float, fs: float, aa: float) -> int:
    """ Estimamos el orden necesario de filtro, suponiendo que ser un Butterworth,
        pues la pendiente es una funcin lineal del orden empleado.
        @param fp: Frecuencia de banda de paso
        @param fs: Frecuencia de muestreo
        @param aa: Atenuacin deseada
        @return: Orden necesario
    """
    return np.ceil((aa / np.log10((fs / 2) / fp)) / 20)

3. Conversor Sigma Delta¶

Se implementa la conversin sigma delta de primer orden, al cual se puede modelar trabajando en tiempo discreto realizando la transformada Z, y obtener as la forma equivalente del mismo:

De esto, aplicando superposicin y operaciones de sistemas bsicas, se puede obtener que la salida se compone de la siguiente expresin:

\begin{equation} Y(z) = X(z) \cdot z^{-1} + N(z) \cdot (1 - z^{-1}) \end{equation}

De esto resulta evidente observar que la entrada tiene aplicada un filtro pasabajos, mientras que el ruido de cuantizacin tiene aplicado un pasaaltos. Esta es una de las fuertes ventajas que posee este proceso de modulacin ya que garantiza que para un rango acotado de bajas frecuencias la contribucin del ruido de cuantizacin se ve disminuda fuertemente, de esta forma si la frecuencia de oversampling es lo suficientemente grande este rango cae sobre la banda de inters de entrada provocando as que la relacin SQNR se vea aumentada. De esta forma, se puede llegar a una nueva expresin de SQNR para estos conversores:

\begin{equation} SQNR = 6.02dB \cdot (n + 1.5 \cdot log_2(L)) - 3.41dB \end{equation}

Esta expresin es genrica en sentido que incorpora la opcin mencionada anteriormente donde n, L no se encuentran vinculados estrictamente, porque podra bajarse la frecuencia de muestreo a una superior a la de Nyquist.

def sigma_delta(x: np.array) -> np.array:
    """ Se simula el proceso de un conversor Sigma Delta sobre el buffer o array
        de datos pre muestreados en las etapas previas.
        @param x: Arreglo de muestras tomadas de la seal con frecuencia de oversampling
        @return: Devuelve el arreglo de '1' y '0' correspondiente a la cuantizacin del Sigma Delta.
    """
    
    def quantization(xi: float) -> int:
        """ Bloque de cuantizacin, defino convencin de salida del ADC.
            @param xi: Error acumulado, salida del integrador
            @return: Estado '1' o '0'
        """
        return 1 if xi > 0 else 0
    
    def dac(y: int, a: float) -> float:
        """ Bloque de dac, para la comparacin la entrada y acumulacin de error.
            @param y: Estado de salida actual
            @param a: Tensin de referencia del DAC
            @return: Devuelve su conversin a analogico
        """
        return (1 if y == 1 else -1) * a

    # Inicializo vector de salida, sistema relajado!
    out = np.zeros((len(x)))
    
    # Defino el error acumulado en la integral
    xi = 0
    
    for i in range(len(x)):
        # Se calcula la seal de error, resta entre entrada y estado de salida actual
        e = x[i] - (dac(out[i - 1], vref) if i > 0 else 0)
        
        # Acumulo el error
        xi += e
        
        # Genero la salida actual
        out[i] = quantization(xi)
    
    return out

4. FIR: Filtro pasabajos¶

En esta etapa, conceptualmente dentro del DSP, se aplica un filtro pasabajos FIR, con coeficientes unitarios, de longitud dada como $M = 2^{N}$ donde $N$ es la cantidad de bits resultantes del ADC. El objetivo es realizar un promedio de las muestras obtenidas como resultado de la modulacin $\Sigma \Delta$, donde podemos pensar este resultado como la cuantizacin. Esto ltimo resulta de tener en cuenta que:

\begin{equation} y_f(n) \in {0, ..., \frac{i}{2^{N}}, ..., 1}, 0 \leq i \leq M \end{equation}
In [572]:
class FIR:
    """ Se crea un filtro FIR que se implementa como ecuacin de recurrencia, guardando los coeficientes
        de la misma.
        @param coef: Lista, array de coeficientes. Si se omiten deben dejarse espacios en 0.
            y(n) = a0.x(n) + a1.x(n-1) + ... + ai.x(n-i)
                    => coef = [a0, ..., ai]
    """
    def __init__(self, coef):
        # Se guardan los coeficientes
        self._a = coef
    
    def filter(self, x):
        """ Aplica el filtro FIR con los coeficientes configurados en su construccin sobre la seal
            de entrada.
            @param x: Seal de entrada
            @return: Salida filtrada
        """
        y = np.zeros(len(x))
        for i in range(len(y)):
            for j, a_j in enumerate(self._a):
                y[i] += a_j * x[i - j] if (i - j) >= 0 else 0
        return y

# Construyo mi filtro FIR

5. Downsampling¶

Se reduce la tasa de muestreo original a la frecuencia de Nyquist para la banda de paso utilizada, y considerando que inicialmente fue muestreada a una frecuencia superior esto podra implicar peligro de producir aliasing, pero con el uso del filtro FIR se limit la banda para garantizar no ocurra esto.

\begin{equation} y(n) = y_f(n \cdot 2^{N}) \end{equation}
